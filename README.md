# ComfyUI-finder

ComfyUI-finder 是一个 ComfyUI 悬浮式文件管理插件。

## 功能

- 快捷键 `F` 开关悬浮窗
- 浏览 ComfyUI 安装目录内文件
- 基础文件操作：`Copy`、`Paste`、`Delete`、`Upload`、`Download`
- **文件搜索**：支持在 ComfyUI 根目录下递归搜索文件
- 选中文件时右侧预览
  - 图片：显示缩略图 + 文件信息
  - 视频：显示可播放预览 + 文件信息
  - 其他文件：显示文件信息
- 文件信息包含：完整路径（可一键复制）、文件大小
- 双击图片可直接加载到工作流
  - 自动创建 `LoadImage` 节点
  - 若图片不在 `input/`，会先复制到 `input/` 再加载
- 搜索模式下双击文件：跳转到文件所在文件夹并高亮该文件
- 支持按 `日期` / `大小` 排序，并可切换升序或降序
- 窗口支持拖动与缩放
- 日志栏支持上下拖拽调整高度
- 文件类型前缀高亮（`DIR` / `FILE` 不同颜色）

## 安装

### 方式一：在 `custom_nodes` 中使用 Git 克隆

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/JHBOY-ha/ComfyUI-finder.git
```

然后重启 ComfyUI

### 方式二：通过 ComfyUI Manager 安装

1. 打开 **ComfyUI Manager**
2. 选择 **Install via Git URL**
3. 粘贴仓库地址：`https://github.com/JHBOY-ha/ComfyUI-finder.git`
4. 安装完成后重启 ComfyUI

## 按钮功能说明

### 主操作栏（第一排）

| 按钮 | 功能说明 |
|------|----------|
| **Copy** | 复制当前选中的文件或文件夹路径到剪贴板 |
| **Paste** | 将之前复制的文件/文件夹粘贴到当前目录 |
| **Delete** | 删除当前选中的文件或文件夹（永久删除，需谨慎） |
| **Upload** | 上传本地文件到当前目录（支持多文件） |
| **Download** | 下载当前选中的文件到本地（不支持文件夹） |

### 导航栏（第二排）

| 按钮 | 功能说明 |
|------|----------|
| **Up** | 返回上级目录 |
| **Refresh** | 刷新当前目录文件列表 |
| **路径显示** | 显示当前所在路径（如：`ComfyUI root / output`） |
| **Date/Size** | 下拉选择排序方式：按日期或按大小 |
| **Desc/Asc** | 切换排序方向：降序/升序 |

### 搜索栏（第三排）

| 按钮/输入框 | 功能说明 |
|-------------|----------|
| **搜索输入框** | 输入关键词搜索文件，按 Enter 执行搜索 |
| **Search** | 执行搜索，在 ComfyUI 根目录下递归查找匹配的文件 |
| **Clear** | 清除搜索，返回正常目录浏览模式 |

### 文件信息区（右侧预览下方）

| 按钮 | 功能说明 |
|------|----------|
| **📋** | 复制完整文件路径到剪贴板 |

## 使用说明

### 基本操作

- **单击文件**：选中文件，右侧显示预览和文件信息
- **双击文件夹**：进入该目录
- **双击图片**：加载到工作流（自动创建 `LoadImage` 节点）
- **双击视频/其他文件**：无特殊操作

### 搜索功能

1. 在搜索框输入关键词（如：`output`）
2. 按 **Enter** 或点击 **Search** 按钮
3. 搜索结果会显示所有匹配的文件（递归搜索）
4. 单击搜索结果：预览文件并显示完整路径
5. **双击搜索结果**：自动跳转到文件所在文件夹，并高亮该文件
6. 点击 **Clear** 退出搜索模式

### 文件预览

- **图片文件**（jpg/png/webp/gif/bmp/svg/avif）：显示缩略图 + 文件信息
- **视频文件**（mp4/webm/mov/m4v/avi/mkv）：显示可播放视频 + 文件信息
- **其他文件**：仅显示文件信息（完整路径、大小）

## 注意事项

- 所有文件操作都限制在 ComfyUI 根目录内，无法访问上级目录
- 删除为永久删除，请谨慎操作
- 搜索功能只搜索文件，不搜索文件夹
- 视频预览需要浏览器支持的视频编码（推荐 H.264）
